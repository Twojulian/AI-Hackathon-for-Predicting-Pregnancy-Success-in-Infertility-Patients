# exp029 — Stacking (Base: LGB+CAT+XGB → Meta: Ridge)

## 실험 개요

| 항목 | 내용 |
|------|------|
| 실험 번호 | exp029 (+ 029_1 변형 실험 포함) |
| 기반 실험 | exp024 피처 구성 + Optuna 파라미터 |
| 모델 | LGB + CAT + XGB → Ridge 메타 모델 |
| 앙상블 방법 | Stacking (2단계 학습) |
| 검증 방법 | Stratified 5-Fold OOF (Base + Meta 모두) |
| 핵심 질문 | Rank Average 대신 Ridge가 조합 가중치를 학습하면 AUC가 올라갈까? |

---

## 실험 배경 및 목적

exp024~028까지 피처 구성, 인코딩 방식을 바꿔봤지만 exp024(0.74025) 벽을 넘지 못했다. 이번에는 **모델 조합 방식 자체를 바꾸는 방향**으로 접근했다.

기존 Rank Average는 세 모델을 무조건 1/3씩 균등하게 조합하는 고정 방식이다. Ridge 메타 모델은 "LGB이 더 잘 맞는 케이스"와 "CAT이 더 잘 맞는 케이스"를 **데이터로부터 학습**해서 가중치를 동적으로 결정한다.

---

## Stacking 구조

```
[Stage 1]
LGB 예측값 (OOF) ─┐
CAT 예측값 (OOF) ─┼─→ [Stage 2] Ridge 메타 모델 → 최종 예측
XGB 예측값 (OOF) ─┘
```

- Base 모델은 exp024 Optuna 파라미터 그대로 사용
- 메타 모델은 C값(0.001 ~ 10.0) 자동 탐색

---

## 실험 변형: USE_ORIGINAL_FEATURES

| 옵션 | 메타 입력 피처 | 설명 |
|---|---|---|
| `False` (exp029) | OOF 예측값 3개 | 순수 Stacking — Base 모델 예측만 조합 |
| `True` (exp029_1) | OOF 예측값 3개 + 원본 피처 102개 = 105개 | 메타 모델이 원본 피처도 직접 참고 |

**False가 맞는 이유:** 원본 피처는 이미 Base 모델들이 충분히 소화했다. 메타 모델에 다시 넣으면 102개 고차원 피처에서 노이즈를 학습하는 **차원의 저주**가 발생한다.

### exp029 (USE_ORIGINAL_FEATURES=False)

| 지표 | 값 |
|---|---|
| Base LGB OOF AUC | 0.73993 |
| Base CAT OOF AUC | 0.73994 |
| Base XGB OOF AUC | 0.73997 |
| Ridge Stacking (C=0.001) | 0.74024 |

| 최종 선택 | Rank Average 0.74025 |

| 최종 선택 | Rank Average 0.74025 |
| vs exp024 | ±0.00000 |

## 결론 및 인사이트

**Ridge Stacking이 Rank Average를 넘지 못했다.** 원인은 두 가지다.

첫째, C값(0.001 ~ 10.0) 전부 0.74024로 동일했다. LGB/CAT/XGB 세 모델의 예측값이 너무 비슷해서 Ridge가 학습할 패턴 자체가 없었다. 세 모델이 거의 같은 말을 하고 있으니 "누굴 더 믿어야 해?"를 학습할 수가 없는 상황이었다.

둘째, USE_ORIGINAL_FEATURES=True는 오히려 0.69로 폭락했다. 차원의 저주가 원인이다.

→ 앙상블 조합 방식 변경으로는 exp024 돌파가 어려운 상황. **다음 방향: ExtraTrees 추가로 앙상블 다양성 확보 시도 (exp030).**
