# exp028 — Target Encoding + RobustScaler

## 실험 개요

| 항목 | 내용 |
|------|------|
| 실험 번호 | exp028 |
| 기반 실험 | exp026 피처 구성 + exp024 Optuna 파라미터 |
| 모델 | LightGBM + CatBoost + XGBoost 앙상블 |
| 앙상블 방법 | Rank Average |
| 검증 방법 | Stratified 5-Fold OOF |
| 핵심 질문 | LabelEncoder 대신 Target Encoding을 쓰면 AUC가 올라갈까? |

---

## 실험 배경 및 목적

exp024~027까지 피처 추가/제거 방향으로 실험했지만 exp024(0.74025) 벽을 넘지 못했다. 피처 구성이 아닌 **인코딩 방식 자체를 바꾸는 새로운 방향**으로 접근했다.

범주형 변수를 단순 정수로 매핑하는 LabelEncoder 대신, **타겟 평균으로 인코딩하는 Target Encoding**을 OOF 방식으로 적용했다. 추가로 수치형 피처에 RobustScaler를 적용해 이상치 영향을 줄였다.

---

## 변경 사항 (vs exp026)

| 항목 | exp026 | exp028 |
|---|---|---|
| 범주형 인코딩 | LabelEncoder | OOF Target Encoding (smoothing=10) |
| 수치형 스케일링 | 없음 | RobustScaler |
| 파라미터 | exp024 Optuna | exp024 Optuna (동일) |
| 피처 수 | 113개 | 113개 (동일) |

## 결론 및 인사이트

Target Encoding이 오히려 하락했다. 원인은 세 가지로 분석된다.

첫째, **트리 모델에는 LabelEncoder가 더 적합하다.** LGB/CAT/XGB는 값의 크기가 아닌 분기점만 보기 때문에, 타겟 평균값으로 변환해도 추가 이득이 없다.

둘째, **CatBoost는 내부적으로 이미 Target Encoding을 수행한다.** 외부에서 한 번 더 적용하면 중복 처리가 된다.

셋째, **RobustScaler는 트리 모델에 불필요하다.** 트리는 피처 스케일에 무관하게 작동한다.

→ 인코딩 방향은 유효하지 않음. 다음 방향으로 **Stacking(메타 모델)** 시도.
