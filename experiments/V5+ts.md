# 🏥 난임 시술 임신 성공 예측 프로젝트 (V5)

난임 시술 환자의 임상 데이터를 기반으로 **임신 성공 여부를 예측**하는 이진 분류 머신러닝 프로젝트입니다.  
EDA 인사이트를 반영한 다양한 파생 변수와 **피처 수 최적화 스윕**을 통해 성능을 단계적으로 개선했습니다.

---

## 📁 파일 구조

```
├── 난임_프로젝트_V5.ipynb            # 메인 노트북
├── train.csv                         # 학습 데이터
├── test.csv                          # 테스트 데이터
├── baseline_RFC_v3.csv               # 전체 피처 모델 제출 파일
├── baseline_RFC_top20.csv            # Top 20 피처 모델 제출 파일
├── baseline_RFC_top{N}.csv           # 최적 N 피처 모델 제출 파일
├── feature_importance_v3.png         # 전체 피처 중요도 (Top 20 시각화)
├── feature_importance_top20.png      # Top 20 재학습 후 피처 중요도
└── auc_by_n_features.png             # 피처 수별 AUC 스윕 그래프
```

---

## 🎯 문제 정의

- **태스크**: 이진 분류 (임신 성공: 1 / 실패: 0)
- **평가 지표**: AUC-ROC
- **타겟 변수**: `임신 성공 여부`

---

## 🔧 전처리 파이프라인

### Step 1. 고결측 컬럼 제거

EDA에서 결측률 84% 이상으로 확인된 컬럼을 제거합니다.

| 컬럼 | 결측률 |
|-----|------|
| 착상 전 유전 검사 사용 여부 | 90.9% |
| PGD 시술 여부 | 99.2% |
| PGS 시술 여부 | 99.2% |
| 난자 해동 경과일 | 99.4% |
| 임신 시도 또는 마지막 임신 경과 연수 | 96.3% |
| 배아 해동 경과일 | 84.2% |

### Step 2. 경과일 변수 — 결측 플래그 생성 후 중앙값 대체

EDA에서 결측 패턴별 임신 성공률이 **2%~39%로 극단적 차이**를 보임 → 결측 여부 자체를 피처로 활용합니다.

대상 컬럼: `난자 채취 경과일`, `난자 혼합 경과일`, `배아 이식 경과일`

각 컬럼마다 `_결측여부` (0/1) 파생 변수를 생성하고, 원본은 중앙값으로 대체합니다.

### Step 3~4. 결측치 처리

- 수치형 → `0` 대체
- 범주형 → `"Unknown"` 대체

### Step 5. 횟수형 변수 정제

"N회", "6회 이상" 등 문자열 형태 값을 정수로 변환합니다.

| 원본 값 | 변환 결과 |
|--------|--------|
| `"3회"` | `3` |
| `"6회 이상"` | `6` |
| `NaN` / `"Unknown"` | `0` |

### Step 6. 나이 변수 수치화

| 변수 | 방식 |
|------|-----|
| 시술 당시 나이 | 연령 구간 → 대표 나이 (예: 만35-37세 → 36) |
| 난자 기증자 나이 | 동일 방식 |
| 정자 기증자 나이 | 동일 방식 |

### Step 10. log1p 변환

EDA에서 skewness > 0.75 로 확인된 배아 관련 수치 변수에 로그 변환을 적용해 분포를 정규화합니다.

| 컬럼 | skewness |
|-----|--------|
| 저장된 배아 수 | 3.78 |
| 미세주입에서 생성된 배아 수 | 1.74 |
| 미세주입된 난자 수 | 1.49 |
| 총 생성 배아 수 | 1.19 |
| 파트너 정자와 혼합된 난자 수 | 1.02 |
| 혼합된 난자 수 | 0.99 |
| 수집된 신선 난자 수 | 0.89 |
| 미세주입 배아 이식 수 | 0.74 |

## ✨ 파생 변수 (EDA 기반 전체 목록)

### IVF / DI 시술 관련 (7개)

| 파생 변수 | 계산식 | 의미 |
|---------|------|-----|
| `IVF_DI_시술_합산` | IVF 시술 + DI 시술 | 전체 시술 경험 총량 |
| `IVF_DI_임신_합산` | IVF 임신 + DI 임신 | 전체 임신 경험 총량 |
| `IVF_DI_출산_합산` | IVF 출산 + DI 출산 | 전체 출산 경험 총량 |
| `IVF_시술_비율` | IVF 시술 / (합산 + ε) | 전체 시술 중 IVF 비중 |
| `IVF_임신_비율` | IVF 임신 / (합산 + ε) | 전체 임신 중 IVF 비중 |
| `IVF_출산_비율` | IVF 출산 / (합산 + ε) | 전체 출산 중 IVF 비중 |
| `시술_대비_임신_비율` | 임신 합산 / (시술 합산 + ε) | 과거 시술 대비 임신 성공률 |

### 불임 원인 관련 (1개)

| 파생 변수 | 계산식 | 의미 |
|---------|------|-----|
| `불임_원인_개수` | 10개 원인 컬럼 합산 | 복합 원인 유무 반영 |

> 대상: 난관 질환, 남성 요인, 배란 장애, 여성 요인, 자궁경부 문제, 자궁내막증, 정자 농도/면역/운동성/형태

### 배아 사용 유형 관련 (1개)

| 파생 변수 | 계산식 | 의미 |
|---------|------|-----|
| `배아_사용_조합` | 동결 + 신선 + 기증 여부를 문자열 결합 | 시술 프로세스 유형 구분 (예: "100", "010") |

### 경과일 결측 플래그 (3개)

`난자 채취 경과일_결측여부`, `난자 혼합 경과일_결측여부`, `배아 이식 경과일_결측여부`

## 🤖 모델

**Random Forest Classifier** (3단계로 학습)

```python
RandomForestClassifier(
    n_estimators=100,
    max_depth=10,
    class_weight="balanced",   # 클래스 불균형 대응
    random_state=42,
    n_jobs=-1
)
```

| 단계 | 설명 | 출력 파일 |
|-----|-----|--------|
| 1차 학습 | 전체 피처 사용 | `baseline_RFC_v3.csv` |
| Top 20 재학습 | 중요도 상위 20개 피처만 재학습 후 성능 비교 | `baseline_RFC_top20.csv` |
| 최적 N 스윕 | 피처 수를 10개 간격으로 변화시키며 AUC 최대 지점 탐색 | `baseline_RFC_top{N}.csv` |

---

## 📊 피처 수 최적화 스윕

피처 중요도 순으로 정렬한 뒤, Top N개 피처로만 재학습하며 AUC를 추적합니다.

```
피처 수를 10개 간격으로 증가 → 각 N에서 RF 재학습 → AUC 비교 → 최적 N 선택
```

결과는 `auc_by_n_features.png`로 저장되며, 최적 N에서 최종 제출 파일을 생성합니다.

| `feature_importance_top20.png` | Top 20 재학습 후 피처 중요도 + AUC 비교 |

### 노트북 실행 순서

1. 환경 설정 및 라이브러리 임포트
2. 전처리 함수 정의 (공통 설정값 포함)
3. 데이터 로드 및 전처리 → 파생 변수 생성
4. X/y 분리 및 train/val split (8:2, stratified)
5. 1차 Random Forest 학습 → 검증 성능 출력 + Top 20 시각화
6. Top 20 피처 재학습 → 성능 비교
7. 피처 수 스윕 → 최적 N 탐색 및 최종 제출 파일 생성

## 🔄 이전 버전(V2)과의 차이점

| 항목 | V2 (baseline) | V5 |
|-----|-------------|-----|
| 파생 변수 | IVF/DI 7개 | IVF/DI 7개 + 불임원인 1개 + 배아조합 1개 + 결측플래그 3개 |
| 경과일 처리 | 단순 결측 → 0 대체 | 결측 플래그 생성 후 중앙값 대체 |
| 로그 변환 | 없음 | 우편향 배아 변수 8개 log1p 변환 |
| 피처 최적화 | 없음 | Top 20 재학습 + N 스윕 자동 탐색 |
| 시각화 | Top 10 | Top 20 + AUC 스윕 그래프 |
| 제출 파일 | 1개 | 최대 3개 (v3 / top20 / topN) |
